# מנהל חבילות משאבים — Minecraft Resource Pack Manager

כלי דפדפן עצמאי (ללא שרת) שמאחד חבילות משאבים של מיינקראפט ממקורות שונים לחבילה אחת.

## מה זה עושה
- מכניסים **קישור** ל־Modrinth / CurseForge / PlanetMinecraft, בוחרים גרסה, ומעלים עוד קובצי **ZIP** לפי הצורך.
- הכלי **ממזג הכל לחבילה אחת** (ZIP) עם `pack.mcmeta` תקין לגרסת המיינקראפט שבחרת.
- אפשר ליצור **כמה פרויקטים**, והכול **נשמר בדפדפן** — לא מתאפס ברענון, אפשר לחזור ולערוך בעתיד.
- **סדר עדיפויות**: גוררים מקורות (או ▲▼) — המקור מספר **1 הכי חשוב וגובר** על התחתונים בהתנגשות קבצים.

## הפעלה
פשוט פותחים את `index.html` בדפדפן (דאבל-קליק). אין צורך בשרת או בהתקנה.

> `vendor/jszip.min.js` מגיע יחד עם הכלי, כך שהוא עובד גם אופליין. `node_modules/` אינו נדרש (הוא רק לצורכי פיתוח/בדיקות).

## מה עובד אוטומטית ומה ידני
| מקור | הורדה אוטומטית | הערה |
|---|---|---|
| **Modrinth** | ✅ מלא | בחירת גרסה + הורדה ישירות בדפדפן (ה־API תומך CORS). |
| **CurseForge** | ⚠️ ידני | אין CORS/מפתח API בדפדפן — הדבק קישור לתיעוד והעלה את ה־ZIP שהורדת. |
| **PlanetMinecraft** | ⚠️ ידני | אין API — הדבק קישור והעלה את ה־ZIP. |
| **קובץ ZIP** | ✅ | העלאה ישירה, נשמר בדפדפן (IndexedDB). |

## איך המיזוג עובד
- סדר המקורות = עדיפות מחסנית (stack) כמו במיינקראפט: **העליון גובר**.
- קבצים חופפים נלקחים מהמקור בעדיפות הגבוהה; הכלי מציג דו"ח התנגשויות.
- תיקיית עטיפה מיותרת בתוך ה־ZIP מזוההת ומקוצצת אוטומטית.
- `pack.mcmeta` נוצר מחדש לפי גרסת היעד (טבלת `pack_format` מלאה מ־1.6 ועד 26.2); `pack.png` נלקח מהמקור העליון שיש לו אחד.
- אפשר גם לערוך ידנית את מספר ה־`pack_format` בכל פרויקט אם צריך ערך שלא ברשימה.
- קבצי `sounds.json`/שפה חופפים מסומנים בנפרד (מיינקראפט לוקח את הקובץ השלם מהעליון — אם צריך מיזוג-תוכן, בדוק ידנית).

## אחסון
- הגדרות הפרויקטים ב־`localStorage`.
- בייטים של ZIPים שהועלו ב־`IndexedDB` (שורדים רענון).
- שים לב: הנתונים נשמרים **פר-דפדפן/פר-מיקום-קובץ**. מעבר למחשב אחר → ראה ייצוא/ייבוא למטה.

## מעבר בין מחשבים — ייצוא / ייבוא פרויקט
- בכל פרויקט יש כפתור **⬇️ ייצוא פרויקט (JSON)** — מוריד קובץ `.rpmproject.json` יחיד שמכיל:
  - כל הגדרות הפרויקט (שם, גרסת יעד, `pack_format`, תיאור).
  - כל המקורות **בסדר העדיפות המדויק שלהם** ומצב הפעלה/כיבוי.
  - עבור קבצי ZIP שהעלית — **הבייטים המלאים מוטמעים בתוך ה־JSON** (בקידוד base64), כך שכל טקסטורה נשמרת ואין צורך להעלות שוב.
  - עבור מקורות Modrinth — נשמר רק הקישור/מזהה הגרסה (הקובץ יורד מחדש אוטומטית בעת הבנייה, אין צורך להטמיע אותו).
- במחשב החדש: לוחצים על **📥** בסרגל הצד ובוחרים את קובץ ה־JSON — הפרויקט נוצר מחדש במלואו, כולל הסדר וכל קובץ שהועלה.
- אימתתי בבדיקות אוטומטיות: הבייטים, הסדר, מצב הפעלה/כיבוי וההגדרות שורדים ייצוא ואז ייבוא לאחר "מחיקת" האחסון המקומי (סימולציה של מעבר מחשב).
